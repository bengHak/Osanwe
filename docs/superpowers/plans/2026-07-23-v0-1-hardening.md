# Osanwe v0.1 Hardening Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make the project-local file-bus flow the only supported architecture and fix the audited release, startup, status, configuration, and shutdown failures.

**Architecture:** Keep `cli -> onboard/project -> session_launch -> zellij/process` and delete the closed legacy module graph. Pass role prompts through provider argv, render the board from current filesystem state, and keep external-command decisions small and directly tested.

**Tech Stack:** Rust 2021, Tokio, Clap, Serde/TOML/JSON, Ratatui/Crossterm, UUID, Zellij CLI, POSIX shell, GitHub Actions.

## Global Constraints

- Preserve the user's pre-existing uncommitted `README.md` changes; never stage the whole file.
- Keep Linux, macOS, and WSL support and Zellij 0.44+ behavior.
- Remove hidden-command compatibility rather than feature-gating it.
- Add no dependencies.
- Write each behavioral regression test first and observe the expected failure.
- Work inline in the current checkout because the user requested no intermediate questions.
